# 2Dgame_1_new

2026/1/3から開発中の”イージーなんていわせない”です
C++ と raylib で制作している、2D横スクロールアクションゲームです。
通常のプレイだけでなく、ゲーム内にステージエディタを入れており、
ステージを作る、保存する、読み込む、すぐにテストプレイする、という流れができます。

GitHub Releases に遊ぶ用ZIPを置いているので、遊ぶだけなら基本的にVisual Studioやraylibの準備は不要です。


## 遊ぶ場合

右側の Releases から最新版をダウンロードしてください。

1. GitHub の右側にある Releases を開く
2. 最新版の「遊ぶ用」を開く
3. Assets にある遊ぶ用ZIPをダウンロードする
4. ZIPを右クリックして「すべて展開」する
5. 展開したフォルダの中にある exe ファイルを起動する

注意：ZIPを展開せずに中から直接起動すると、画像や音声を読み込めない場合があります。
必ず一度フォルダとして展開してから起動してください。


## 開発用として動かす場合

Code からダウンロードしたものは開発用です。
こちらをビルドして動かす場合は、Visual Studio と raylib が必要になる場合があります。

1. Visual Studio を開く
2. `2Dgame_1.slnx` または `2Dgame_1.vcxproj` を開く
3. raylib が使える状態になっていることを確認する
4. ビルドして実行する

遊ぶだけの場合は、Code からではなく Releases の遊ぶ用ZIPを使ってください。



## 使用技術

- C++20
- raylib
- Visual Studio / MSVC
- CSV・JSONによるステージ保存
- PNGテクスチャ、MP3 BGM、WAV SE
- 独自ステージエディタ

ゲームエンジンは使用せず、プレイヤー制御、当たり判定、カメラ、敵、アイテム、ステージギミック、描画、音声、エディタをC++で実装しています。

## 主な特徴

### 2Dアクション

- 左右移動、ジャンプ、重力処理
- X軸・Y軸を分離した当たり判定
- 敵への踏みつけと側面接触の判定
- トゲ、落下、敵、罠、おじさんパンチなど複数の死亡原因
- リスポーンと死亡回数の表示
- 死亡アニメーションと死亡画面
- 通常ステージと縦長ステージに対応したカメラ制御

### おじさん演出

- 画面右下におじさんをUIとして表示
- 死因・ステージ・死亡回数に応じて台詞を変更
- 敵ごとの専用台詞キーに対応
- コメントブロックから任意のメッセージを表示
- `@キー名`による外部台詞参照
- パンチトリガーエリアによる専用死亡演出

台詞は `ojisan_lines.text` に次の形式で記述します。

```text
death_fall|また落ちたのか
death_spike_stage2|そのトゲ、さっきも見ただろう
death_enemy_walker_patrol|正面から行くからだ
```

### アイテム

- ジャンプ力上昇
- 移動速度上昇
- 無敵
- 過剰ジャンプ
- 移動・ジャンプ能力を下げるデバフ

アイテム効果は時間制で管理され、複数の効果状態を `ItemManager` が更新します。

### 敵

現在、次の4種類を管理できます。

| 敵タイプ | 概要 |
|---|---|
| `WALKER` | 地上を巡回する敵 |
| `FLYER` | 空中を移動する敵 |
| `SHOOTER` | 射撃用パラメータを持つ敵 |
| `JUMPCOPY` | ジャンプ関連パラメータを持つ敵 |

エディタでは敵の種類、位置、速度、巡回距離などを個別に設定できます。

| 操作 | 内容 |
|---|---|

| `Enter` | 選択したステージを開始 |
| `V / F1` | 選択中ステージをエディタで開く |

### プレイ中

| 操作 | 内容 |
|---|---|
| `← / →` または `A / D` | 左右移動 |
| `Space / W / ↑` | ジャンプ |
| `R` | 近くのばねを拾う／持っているばねを置く |
| `Tab` | ポーズメニュー |

| 出口ドア付近で `W / ↑` | 次のステージへ移動 |
| 死亡画面で `Enter / Space` | リスタート |

エレベーター上では、ジャンプ入力は `Space` が使用されます。

### ポーズメニュー

| 操作 | 内容 |
|---|---|

| `Enter / Space` | 決定 |
| `Tab / Esc` | ゲームへ戻る |

## ステージエディタ

ゲーム本編と同じ実行ファイル内にエディタを搭載しています。配置内容は `PlacedObject` と `PlacedEnemy` に保存され、プレイ時に `Stage` と `EnemyManager` へ変換されます。

### 基本操作

| 操作 | 内容 |
|---|---|
| 左クリック | オブジェクトまたは敵を配置 |
| 右クリック | カーソル下のオブジェクトを削除 |
| 中クリックドラッグ | カメラ移動 |
| マウスホイール | ツールバー上では種類変更、それ以外ではズーム |
| `T` | カーソル下のオブジェクトを選択し、プロパティを表示 |
| `Delete` | 選択中オブジェクトまたは敵を削除 |
| `Tab` | オブジェクト種類を次へ送る |
| `1`〜`9` | 先頭9種類を直接選択 |
| `Q / E` | 新規配置サイズの横幅を縮小／拡大 |
| `W / S` | 新規配置サイズの高さを拡大／縮小 |
| `R` | グリッドサイズを `10 → 20 → 40 → 80` の順で変更 |
| `Ctrl + Z` | 直前に削除したオブジェクトを復元 |
| `Ctrl + S` | CSVとJSONを保存 |
| `Ctrl + L` | CSVを再読み込み |
| `C` | 保存・ステージへ反映してプレイテスト開始 |
| `V / F1` | 保存してゲームへ戻る |

### 選択中オブジェクトの見た目編集

| 操作 | 内容 |
|---|---|
| `[` / `]` | 前／次の `SpriteId` に切り替え |
| `H` | 左右反転 |
| `J` | 上下反転 |
| `K / L` | 反時計回り／時計回りに90度回転 |
| `- / =` | 選択中オブジェクトの幅と高さを10pxずつ変更 |
| `P` | コピー済みオブジェクトを30pxずらして貼り付け |

`StageEditor.cpp` では `C` が選択中オブジェクトのコピーにも割り当てられていますが、`main.cpp` では同じ `C` がプレイテスト開始に使われています。そのため、現状はキー割り当てが競合しています。

### パラメータ編集

`T` で選択したオブジェクトの右側パネルから、速度、移動距離、遅延、方向、検知範囲、遷移先などを編集できます。

- 数値項目：クリック後に数値入力し `Enter` で確定
- 真偽値項目：クリックでON/OFF切り替え
- `Esc`：編集中の入力をキャンセル
- コメント：日本語入力または `Ctrl + V` による貼り付けに対応

## ステージ保存

保存先はゲームルートの次のディレクトリです。

```text
data/stages/
```

通常ステージは次の名前で保存されます。

```text
data/stages/stage_edit_1.csv
data/stages/stage_edit_2.csv
data/stages/stage_edit_3.csv
...
```

エディタ専用ステージは次の名前です。

```text
data/stages/stage_editor_output.csv
```

保存済みCSVが存在する場合、固定配置の `StageInit_*` より後に読み込まれ、エディタ内容でステージを上書きします。

### CSV形式

通常オブジェクトは次の列で保存されます。

```text
type,typeId,x,y,w,h,p0,p1,p2,p3,p4,p5,text,spriteId,rotation,flipX,flipY
```

敵は `[ENEMIES]` セクション以降に保存されます。

```text
enemyTypeId,posX,posY,p0,p1,p2,p3,p4,p5
```

古いCSVにスプライト列が存在しない場合は、`SpriteId::None`、回転0度、反転なしとして読み込む後方互換処理があります。

JSONは保存できますが、JSONからの読み込みは現在未実装です。ゲームでの標準保存・読み込みにはCSVを使用します。

## スプライト管理

`EditorObjectType` と `SpriteId` を分離しています。

```text
EditorObjectType = 当たり判定・ギミックの挙動
SpriteId         = 描画する画像
```

この構造により、普通床の当たり判定を維持したまま、金属床、看板、土管、ドア、NEXT STAGEロゴなどへ見た目だけを変更できます。

スプライトは `SpriteDatabase.cpp` のテーブルで、名前と画像パスを対応付けています。新しい画像を追加する場合は、主に次の2か所を更新します。

1. `SpriteDatabase.h` の `SpriteId`
2. `SpriteDatabase.cpp` の `g_spriteTable`
