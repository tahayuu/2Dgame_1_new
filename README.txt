

## 作品名
2Dgame_1（2D横スクロールアクション + ステージエディタ）

*概要
C++ と raylib で制作した2Dアクションゲームです。  
ゲーム本編とステージ編集機能が同じ実行ファイル内にあり、`V` または `F1` で切り替えて、配置・調整・保存・再プレイを行えます。

## 使用技術
- C++
- raylib
- Visual Studio
- CSV / JSON（ステージ保存）

## 操作方法

### タイトル/ステージ選択
- `Space`: ステージ選択開始
- `↑ / ↓`: ステージ選択
- `Enter`: 選択ステージ開始
- `V` または `F1`: 選択ステージをエディタで開く

### プレイ中
- `← / →` または `A / D`: 移動
- `Space`（通常時は `W` / `↑` も可）: ジャンプ
- `Tab`: ポーズ
- `V` または `F1`: エディタへ切り替え
- `R`: 特定のアイテムの保管/設置
- 死亡後 `Enter` または `Space`: リスタート

### エディタ中
- マウス左クリック: オブジェクト配置
- マウス右クリック: オブジェクト削除
- マウス中ドラッグ: カメラ移動
- マウスホイール:  
  - ツールバー上: オブジェクト種変更  
  - それ以外: ズーム
- `T`: カーソル下オブジェクト選択（プロパティ編集対象）
- `Delete`: 選択オブジェクト削除
- `Ctrl + Z`: 取り消し
- `Ctrl + S`: 保存（CSV/JSON）
- `Ctrl + L`: CSV読込
- `Tab`: オブジェクト種を順送り
- `1`〜`9`: 一部オブジェクト種の直接選択
- `Q / E`: 配置サイズ（幅）変更
- `W / S`: 配置サイズ（高さ）変更
- `R`: グリッドサイズ切替
- `V` または `F1`: プレイに戻る
- `C`: 保存→ステージ反映→プレイテスト開始

## 実装した機能

### ゲーム進行
- ゲーム状態管理（START / PLAYING / EDITOR / PAUSE / DEAD）
- リスポーン、死亡画面、死亡回数表示
- ステージ選択（6項目）

### ステージギミック（エディタ配置対応）
`EditorObjectType` として 49 種類が定義されています。  
例:
- 各種床・トラップ・移動床
- ジャンプ台、砲台、磁石、追尾/回転系ハザード
- コメントブロック、リスポーン、出口ドア
- ばね（Spring）
- クレーン発射台 / クレーン
- おじさんパンチトリガーエリア

### クレーン関連
- 発射台接触で上方向に射出
- クレーンがプレイヤーを掴んで運搬する処理
- 条件成立でクレーン由来の死亡

### おじさん演出
- 画面右下におじさんUIを描画（カメラ非依存）
- パンチトリガーエリア侵入時にパンチ死亡演出
- 死亡画面でパンチエフェクト重ね描画

### ダイアログ
- `ojisan_lines.text` 読み込み
- コメントブロックからメッセージ表示
- `@キー` 形式でダイアログキー参照

## ステージ編集画面（詳細）
このプロジェクトのエディタは、`StageEditor` 上の `objects` 配列を編集し、`EditorExportToStage` で実行用 `Stage` に変換する構成です。

- **配置情報**  
  - 種別（type）
  - 矩形（x, y, w, h）
  - 汎用パラメータ（p0〜p5）
  - 文字列（コメント用text）
- **保存形式**  
  - CSV: `type,typeId,x,y,w,h,p0..p5,text`
  - JSON: 簡易オブジェクト一覧
- **入出力**  
  - `EditorImportFromStage`: 既存ステージ → エディタ
  - `EditorExportToStage`: エディタ → 実行ステージ
- **編集UI**  
  - ツールバー（種別選択）
  - プロパティパネル（数値/真偽値/コメント編集）
  - グリッドスナップ、ズーム、ドラッグ移動

## 工夫した点
- 実行用データ（`Stage`）と編集用データ（`PlacedObject`）を分離し、相互変換を実装
- 配置オブジェクトをCSVで保存し、ステージごとの編集データを再利用可能にした
- コメントブロックに自由文とダイアログキー参照（`@...`）の両方を持たせた
- プレイ中とエディタ中で入力系を分離し、即時プレイテストしやすい流れにした

## 実行方法
1. Visual Studioでソリューションを開く  
2. 一番下部分にある2Dgame_1.slnxをダブルクリック
3. ビルドして実行（`F5` または `Ctrl+F5`）  
4. 起動後、`Space` でステージ選択 → `Enter` で開始  

## 指針
ギミックをこれからも多く作りマリオメーカーのような体制で自由にステージが作れるようにしていきたい。
できるだけプレイヤーが腹の立つギミックを納得性のある形で実装したい。
例えばカーソルで動かせる床があるのですがその床に気づいてもらえる前にカーソルで動かせるスイッチを挟むことでプレイヤーに｛このゲームはカーソルにも対応しているのか｝と気づかせてから床が実は動かせることに気づいてもらうなどところどころに理不尽を納得させるようなゲーム性にしていきたい。

私の考えるプレイヤーにとって面白いとは達成感と没入感が基盤であると考えている。適度な壁を提供しプレイヤーが直感的に操作でき統一された世界観の中でクリアを目指していく過程が人は面白いと感じると私は思っている。
