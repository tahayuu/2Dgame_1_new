# 2Dgame_1_new

C++ と raylib で制作している、2D横スクロールアクションゲームです。
通常のプレイだけでなく、ゲーム内にステージエディタを入れており、
ステージを作る、保存する、読み込む、すぐにテストプレイする、という流れができます。

GitHub Releases に遊ぶ用ZIPを置いているので、遊ぶだけなら基本的にVisual Studioやraylibの準備は不要です。


## 遊ぶ場合

右側の Releases から最新版をダウンロードしてください。

1. GitHub の右側にある Releases を開く
2. 最新版の「遊ぶ用」を開く
3. Assets にある遊ぶ用ZIPをダウンロードする
4. ZIPを右クリックして「すべて展開」する
5. 展開したフォルダの中にある exe ファイルを起動する

注意：ZIPを展開せずに中から直接起動すると、画像や音声を読み込めない場合があります。
必ず一度フォルダとして展開してから起動してください。


## 開発用として動かす場合

Code からダウンロードしたものは開発用です。
こちらをビルドして動かす場合は、Visual Studio と raylib が必要になる場合があります。

1. Visual Studio を開く
2. `2Dgame_1.slnx` または `2Dgame_1.vcxproj` を開く
3. raylib が使える状態になっていることを確認する
4. ビルドして実行する

遊ぶだけの場合は、Code からではなく Releases の遊ぶ用ZIPを使ってください。

## 2Dgame_3 で追加した主な内容

### ステージエディタ上でスプライトを反映

2Dgame_3 では、エディタに置いたオブジェクトに画像を割り当てられるようにしました。
今までは床やブロックなどを仮の四角形として表示していましたが、
`SpriteId` を設定することで、見た目だけを画像に変更できます。

ギミックとしての種類と、見た目としての画像を分けているので、
同じ床でも画像だけを変えることができます。

例：

- MetalFloor01
- MetalFloor02
- MetalWall01
- MetalWall02
- Ladder01
- PipeStraight
- PipeCorner
- Vent
- Door
- Fence
- Beam

画像は主に `assets/images/stage/stage_3/` に入っています。

### スプライトの保存・読み込み

CSV保存時に、通常の配置情報に加えて以下の見た目情報も保存します。

- spriteId
- rotation
- flipX
- flipY

そのため、エディタで画像を選んだ後に保存しても、
次に読み込んだときに画像、回転、反転の設定が残ります。

CSVの形式は以下のようになっています。

```csv
type,typeId,x,y,w,h,p0,p1,p2,p3,p4,p5,text,spriteId,rotation,flipX,flipY
```

古いCSVに `spriteId` などがない場合でも、読み込み時は見た目なしとして扱うようにしています。

### エディタ上での画像切り替え操作

オブジェクトを選択した状態で、以下のキーを使うと見た目を調整できます。

| キー | 内容 |
|---|---|
| [ / ] | スプライトを前後に切り替え |
| H | 左右反転 |
| J | 上下反転 |
| K | 反時計回りに90度回転 |
| L | 時計回りに90度回転 |

## 注意点

- 遊ぶだけなら Releases の遊ぶ用ZIPを使ってください。
- Code から落としたものは開発用なので、環境によっては raylib の準備が必要です。
- ZIPは必ず展開してから起動してください。
- exe の近くに assets や data がないと、画像、音声、ステージデータを読み込めません。
- Visual Studio の一時ファイルや Debug フォルダは、開発途中のファイルです。


## 制作で意識していること

マリオメーカーのように、自由にステージを作れる2Dアクションゲームを目指しています。

ただ難しいだけではなく、プレイヤーが「なぜ失敗したのか」を理解できるようにして、
理不尽さの中にも納得感があるギミックにしたいと考えています。

たとえば、カーソルで動かせる床をいきなり出すのではなく、
先にカーソル操作に気づけるスイッチを置くことで、
プレイヤーが自然に仕組みを理解できるような作りを目指しています。
